# MSKTool

This workspace preserves the original `ADTMCplus` browser tool under [legacy/index.html](/Users/matthewholtkamp/Documents/ADTMCplus-msk/legacy/index.html) and adds a deterministic MSK screening and referral application that can run directly from the repo root [index.html](/Users/matthewholtkamp/Documents/ADTMCplus-msk/index.html) on GitHub Pages.

## Structure

- `index.html`: GitHub Pages entrypoint for the static MSK tool
- `static/app.js` and `static/styles.css`: browser-ready bundle and stylesheet generated by `npm run build`
- `legacy/index.html`: original single-file SOAP generator preserved intact
- `apps/msk-referral`: source React UI used for both local authoring and the static bundle
- `packages/msk-content`: validated pathway content, shared types, schemas, and deterministic rule engine

## Run

```bash
npm install
npm run build
npm run preview:pages
```

Then open [http://localhost:3000](http://localhost:3000).

For source-level React development you can still use:

```bash
npm run dev
```

The build script emits a root `index.html`-compatible static bundle so the same app can be served by GitHub Pages without any server runtime.

## Notes

- The MSK tool is strictly deterministic. There is no LLM or AI dependency in the clinical workflow.
- Every rule, question, escalation path, and note section is derived from the encoded PDF algorithms.
