# MSKTool

This workspace preserves the original `ADTMCplus` browser tool under [legacy/index.html](/Users/matthewholtkamp/Documents/ADTMCplus-msk/legacy/index.html) and now presents the MSK pathways in an `ADTMC`-style single-page triage interface at [index.html](/Users/matthewholtkamp/Documents/ADTMCplus-msk/index.html).

## Structure

- `index.html`: root single-page triage UI modeled after the `ADTMC` repo structure
- `static/app.js`: browser-ready deterministic logic bundle generated by `npm run build`
- `legacy/index.html`: original single-file SOAP generator preserved intact
- `apps/msk-referral`: retained source material and tests from the first implementation pass
- `packages/msk-content`: validated pathway content, shared types, schemas, and deterministic rule engine

## Run

```bash
npm install
npm run build
npm run preview:pages
```

Then open [http://localhost:3000](http://localhost:3000).

## Notes

- The MSK tool is strictly deterministic. There is no LLM or AI dependency in the clinical workflow.
- The root app intentionally mirrors the `ADTMC` repo interaction model: home/search, master red flags, protocol-by-protocol triage, and a copyable encounter summary.
- Every rule, question, escalation path, and note section is derived from the encoded PDF algorithms.
